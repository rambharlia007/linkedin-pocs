How does context flow into a subagent? I'd been using them for months and never thought to check.

Then I noticed my code-reviewer subagent kept asking for the file I had just been editing. The parent had it. The subagent didn't.

That set me off.

Three runs. Same buggy C# file. Same model.

Run 1: focused reviewer, vague brief, "review my recent change."
It refused. Asked for the file path. 27K tokens on the refusal.

Run 2: general-purpose subagent, same vague brief.
It went exploring. Git, glob, opened the file itself. 38K tokens, 45 seconds doing what I already knew.

Run 3: same general-purpose subagent, briefed properly. File path, contents, focus areas, all in the prompt.
14 seconds. 28K tokens. Nine clean findings.

The bit I had never internalized: a subagent doesn't inherit anything from the parent's session.

Not the conversation. Not the files just opened. Not the question you asked two turns back.

This isn't a Claude Code thing. LangGraph nodes work the same way. CrewAI agents too. Anyone spawning a subagent via a fresh model call does. It's the architectural default.

Which means the contract is one-way. Whatever goes into the delegation prompt is what they have. The rest stays in your head.

Most of the time my subagent looks broken, it's because I briefed it badly. Not because the model is dumb.

Anyone else found something inside agent orchestration that you wish you'd noticed sooner?
